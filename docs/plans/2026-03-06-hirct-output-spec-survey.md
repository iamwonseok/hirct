> 목적: HIRCT 각 emitter의 입출력, Golden Output 기준, IR 요구사항, 미해소 Gap을 단일 문서로 정의한다.

## 1. 전체 구조

### 1.1 파이프라인

```
RTL (.v/.sv)
  ↓
CIRCT MLIR IR (hw / comb / seq 다이얼렉트)
  ↓
┌──────────────────────────────────────────────────────────────────────┐
│  HIRCT lowering pipeline                                             │
│  sim_cleanup → unroll_process_loops → RemoveControlFlow              │
│  → Canonicalizer → process_flatten → process_deseq                  │
│  → signal_lowering → CSE → Canonicalizer                            │
└──────────────────────────────────────────────────────────────────────┘
  ↓
┌─ GenModel    → cmodel/{mod}.h + .cpp       (C++ cycle-accurate 모델)
├─ GenVerify   → verify/verify_{mod}.cpp     (RTL vs CModel 비교 드라이버)
├─ GenFuncModel→ func_model/{mod}.h + .c     (기능 추상화 C 모델)
├─ GenTB       → tb/{mod}_intf.sv + _tb.sv   (SV 테스트벤치 + interface)
├─ GenDPIC     → dpi/{mod}_dpi.h + .cpp + .sv (DPI-C 래퍼)
├─ GenRAL      → ral/{mod}_ral.sv + _hal.h + _drv.c (레지스터 맵)
├─ GenDoc      → doc/{mod}.md                (하드웨어 문서)
└─ GenCocotb   → cocotb/test_{mod}.py        (cocotb 테스트 스캐폴드)
```

### 1.2 Emitter 요약표

| Emitter | 입력 | 출력 | 소비자 | 독립? | 우선순위 |
|---------|------|------|--------|-------|---------|
| **GenModel** | CIRCT IR | `cmodel/*.h/.cpp` | FW팀, 검증팀 | 독립 (핵심) | **P0** |
| **GenVerify** | CIRCT IR + CModel | `verify/verify_{mod}.cpp` | CI, 검증팀 | GenModel 의존 | **P0** |
| **GenFuncModel** | CIRCT IR | `func_model/*.h/.c` | 사내 C sim 팀 | 독립 | P1 |
| **GenTB** | CIRCT IR | `tb/*.sv` | 검증팀 | 독립 | P1 |
| **GenDPIC** | CIRCT IR + CModel | `dpi/*.h/.cpp/.sv` | 검증팀 (VCS/Questa) | GenModel 의존 | P1 |
| **GenRAL** | CIRCT IR | `ral/*` | FW팀, 검증팀 | 독립 | P1 |
| **GenDoc** | CIRCT IR | `doc/*.md` | 전체 팀 | 독립 | P2 |
| **GenCocotb** | CIRCT IR | `cocotb/test_*.py` | 검증팀 | 독립 | P2 |

### 1.3 C++ 모델 연결 경로

| | GenVerify | GenDPIC |
|---|-----------|---------|
| **시뮬레이터** | Verilator | VCS / Questa |
| **bridge** | C++ 직접 링크 (같은 프로세스) | DPI-C (`import "DPI-C"`) |
| **C++ 모델 관여** | O | O |

> GenTB/GenWrapper는 C++ 모델과 **무관**. 순수 SV 포트 번들.

## 2. Emitter별 스펙

### 2.1 GenModel — C++ Cycle-Accurate Model

**목적**: RTL과 비트/사이클 동일한 standalone C++17 클래스. `model.step()` 한 줄로 1 클럭 진행.

**Verilator와의 차이**:
- Verilator: 전체 시뮬레이터 (`eval()` + delta-cycle). FW 엔지니어가 사용하기 어려운 API.
- HIRCT GenModel: 추상 C++ 모델 (`step()` = 클럭 엣지 1회). 외부 의존성 없는 standalone.

**API 규약**:

| 함수 | 설명 |
|------|------|
| `do_reset()` | 모든 레지스터 리셋값으로 초기화, 입력 포트 0으로 설정 |
| `step()` | 기본 클럭 1 엣지 진행 (멀티 클럭 시 모든 도메인 호출) |
| `step_<clock>()` | 멀티 클럭: 해당 도메인만 진행 |
| `eval_comb()` | 조합 로직 평가 (입력 변경 후 출력 갱신) |

**포트 타입 매핑**:

| 비트폭 | C++ 타입 |
|--------|---------|
| 1 | `bool` |
| 2~8 | `uint8_t` |
| 9~16 | `uint16_t` |
| 17~32 | `uint32_t` |
| 33~64 | `uint64_t` |
| 65+ | Phase 1: `uint64_t` (truncate, KL-3) / Phase 2: `uint32_t[]` |

**Golden Output 예시 (`cmodel/uart_top.h`)**:

```cpp
// AUTO-GENERATED by hirct-gen — DO NOT EDIT
#ifndef CMODEL_UART_TOP_H
#define CMODEL_UART_TOP_H

#include <cstdint>
#include "DW_apb_uart/cmodel/DW_apb_uart.h"

class uart_top {
public:
  // Ports (직접 접근)
  bool UART_PCLK;
  bool UART_PRESETn;
  bool PSEL;
  bool PENABLE;
  bool PWRITE;
  uint32_t PADDR;    // 24-bit
  uint32_t PWDATA;   // 32-bit
  uint32_t PRDATA;   // 32-bit (output)
  bool UART0_INTR;   // output

  void do_reset();
  void step();
  void step_uart_pclk();   // APB 클럭 도메인 (멀티 클럭)
  void eval_comb();

private:
  DW_apb_uart inst_U_UART0;

  uint32_t reg_ssa_rbr;
  uint32_t next_ssa_rbr;
  // ...
};
#endif
```

**정확성 기준**:
- 동일 입력 시퀀스에서 RTL(Verilator)과 매 사이클 출력 일치
- `do_reset()` 후 모든 레지스터가 RTL posedge reset 후와 동일
- 멀티 클럭 모듈에서 각 도메인 step이 독립적으로 동작

**현재 Gap**:

| Gap | KL | 상태 |
|-----|----|------|
| 65+ 비트 truncate (`uint64_t`) | KL-3 | P1 |
| `llhd.prb/drv` 잔존 모듈 스킵 | KL-5 | P1 |
| FSM `ceq` 기반 모듈 스킵 | KL-10 | P1 |
| `llhd.combinational` 미지원 | KL-11 | P2 |
| `hw.array_inject` 미지원 | KL-16 | P2 |
| 멀티 클럭 step 분리 (clock gate 경로) | — | P2 |
| 트레이스 미지원 (VCD/FST) | — | P3 |

### 2.2 GenVerify — RTL vs CModel 비교 드라이버

**목적**: GenModel이 생성한 C++ 모델과 Verilator로 컴파일한 RTL을 매 사이클 비교.

**동작**: `hirct-verify input.v` → `verify_{mod}.cpp` 생성 → Verilator 빌드 → 10 seed × 1000 cyc 랜덤 자극 비교

**Golden Output 예시 (`verify/verify_uart_top.cpp`, 핵심 구조)**:

```cpp
// AUTO-GENERATED by hirct-gen — DO NOT EDIT
#include "Vuart_top.h"       // Verilator
#include "../cmodel/uart_top.h"  // CModel

int main(int argc, char** argv) {
  Vuart_top rtl;
  uart_top  cmodel;
  // 랜덤 자극 생성 → 양쪽에 동일 입력 → 매 사이클 출력 비교
  for (int cyc = 0; cyc < 1000; ++cyc) {
    // apply random inputs
    rtl.eval(); cmodel.step();
    // compare outputs: PSEL, PRDATA, UART0_INTR ...
    if (cmodel.PRDATA != rtl.PRDATA) {
      fprintf(stderr, "MISMATCH at cycle %d\n", cyc);
      return 1;
    }
  }
  return 0;
}
```

**현재 Gap**:

| Gap | 상태 |
|-----|------|
| 리셋 초기화 순서 (RTL reset 사이클 수 불일치) | P1 |
| SARIF 리포트 출력 | P3 |

### 2.3 GenFuncModel — Functional C Model

**목적**: RTL 동작을 사람이 읽을 수 있는 C 코드로 추상화. 드라이버 뼈대 + 시스템 시뮬레이터 stub.

**GenModel과의 차이**:

| | GenModel | GenFuncModel |
|---|----------|-------------|
| 추상화 | 비트/사이클 정확 | 기능 정확 (동작 의미 보존) |
| 코드 스타일 | 비트 연산, SSA 변수명 | 레지스터 이름, if/switch |
| 용도 | 검증, co-sim | 드라이버 참조, 시스템 sim stub |

**구현 전략 (단계적)**:
1. 레지스터 인터페이스 추출 → 주소 오프셋 `#define` + `read()`/`write()` API
2. FSM 인식 (선택적) → `enum State` + 상태 전이 함수
3. 제어 흐름 추상화 → `if`/`switch` 변환 (장기)

**현재 Gap**:

| Gap | 상태 |
|-----|------|
| 현재 FSM-only 구현 (FSM 없으면 산출물 없음) | 재설계 필요 |
| 주소 디코딩 분석 미구현 | P2 (AddressDecodingAnalysis 신규) |
| 읽기/쓰기 동작 추출 미구현 | P2 (RegisterBehaviorAnalysis 신규) |
| FSM `ceq` 미지원 | KL-13 |

> **전략 방향 변경 (2026-03-08)**: GenFuncModel은 pure C 본체 + 환경별 래퍼(SystemC TLM/DPI-C/QEMU) 아키텍처로 재설계 예정. 모델 본체에 `read()/write()/tick()` API를 제공하고, 래퍼가 대상 환경(가상 플랫폼, VCS/Questa, QEMU 에뮬레이터)에 맞게 포장한다. 상세는 `2026-03-08-hirct-product-strategy.md` Section 5 참조.

### 2.5 GenDPIC — DPI-C Wrapper

**목적**: GenModel의 C++ 클래스를 VCS/Questa SV 시뮬레이터에서 `import "DPI-C"`로 호출할 수 있게 래핑.

**산출물 3파일**:
- `dpi/{mod}_dpi.h`: C 함수 선언 (extern "C")
- `dpi/{mod}_dpi.cpp`: 구현 (void* 핸들 기반 인스턴스 관리)
- `dpi/{mod}_dpi.sv`: SV `import "DPI-C"` 선언 + 래퍼 모듈

**Golden Output 예시 (`dpi/uart_top_dpi.h`, 핵심)**:

### 2.6 GenRAL — Register Abstraction Layer

**목적**: RTL에서 레지스터 맵을 자동 추출하여 UVM RAL (`_ral.sv`), C HAL (`_hal.h`), 드라이버 뼈대 (`_drv.c`) 생성.

**구조적 한계**: 업계 모든 RAL 도구는 SystemRDL/RALF/HJSON 같은 레지스터 기술 언어를 입력으로 요구한다. RTL IR만으로는 필드 경계와 접근 권한을 복원할 수 없다.

**권장 아키텍처 (하이브리드)**:
- `--ral-rdl` 없으면: IR-only (레지스터 존재 여부 + 오프셋 추정만)
- `--ral-rdl=design.rdl` 있으면: SystemRDL 보강 (필드 수준 완전한 RAL)

**현재 Gap**:

| Gap | 상태 |
|-----|------|
| 주소 디코딩 분석 미구현 | P2 |
| 필드 경계/접근 권한 IR에서 복원 불가 (구조적 한계) | 하이브리드 옵션 D 채택 예정 |

## 3. IR 요구사항

### 3.1 GenModel 진입 조건 (Precondition)

**현재 구현된 체크** (`GenModel::emit()`):
```
// llhd.prb / llhd.drv 잔존 여부만 검사 → 있으면 skip
// ProcessOp/WaitOp/HaltOp/YieldOp/CombinationalOp은 허용 (GenModel이 직접 처리)
```

**필요하지만 미구현된 체크**:
- `iN (N>64)` 포트/레지스터 존재 여부 (KL-3)
- `hw.array_inject` 존재 여부 (KL-16)
- `llhd.combinational` 잔존 여부 (KL-11)

### 3.2 필수 정보 → IR Op 매핑

| 필수 정보 | IR 소스 | 현재 확보 | 비고 |
|-----------|---------|----------|------|
| 포트 목록/방향/비트폭 | `hw.module` args | ○ | — |
| 클럭 포트 식별 | 이름 휴리스틱 + `seq.to_clock` | ○ | 이름 의존 fallback 위험 |
| 리셋 포트 + 극성 | 이름 suffix | △ | IR 기반 극성 판단 미구현 |
| 레지스터 (이름/비트폭/리셋값) | `seq.firreg`, `seq.compreg` | ○ | — |
| 클럭 도메인 → 레지스터 매핑 | `seq.*` clock 오퍼랜드 역추적 | △ | internal clock gate 경로 fallback |
| 메모리 (depth/width/RW) | `seq.firmem.*` | ○ | — |
| 조합 로직 DAG | `comb.*` SSA | △ | `hw.array_inject` 미지원 (KL-16) |
| 서브모듈 인스턴스 + 연결 | `hw.instance` | ○ | Kahn's 위상 정렬 사용 |
| 넓은 비트 (65+) | `iN (N>64)` | [X] | truncate → KL-3 |
| FSM 패턴 | `seq.compreg` + `comb.icmp` + `comb.mux` | △ | `ceq` 미지원 KL-13 |
| 레지스터 읽기/쓰기 동작 | 조합 로직 분석 | [X] | RegisterBehaviorAnalysis 신규 필요 |
| 주소 디코딩 → 오프셋 | 조합 로직 분석 | [X] | AddressDecodingAnalysis 신규 필요 |

### 3.3 현재 DAG 순회 지원 op (`EmitExpr.cpp`)

**지원** (`emit_op_expr()`):
- `comb.*`: Add, Sub, Mul, And, Or, Xor, Shl, ShrU, ShrS, Mux, Concat, Extract, Replicate, Parity, ICmp (eq/ne/ult/ule/ugt/uge/slt/sle/sgt/sge + ceq/weq/cne/wne), DivU, DivS, ModU, ModS
- `hw.*`: ConstantOp, BitcastOp, ArrayCreateOp, ArrayGetOp, AggregateConstantOp

**미지원** → `"\x01"` sentinel 반환:
- `hw.array_inject` (KL-16)
- `hw.struct_extract`, `hw.struct_create`
- 기타 미매핑 op

### 3.4 필요한 신규/개선 Pass

| Pass | 목적 | 우선순위 |
|------|------|---------|
| **WideIntLowering** | 65+ 비트 → `uint32_t[]` 배열 코드 생성 | P1 (KL-3) |
| **SignalLoweringV2** | 잔존 `llhd.prb/drv` 완전 제거 | P1 (KL-5) |
| **FSMAnalysisV2** | `ceq` ICmpPredicate 지원, 복수 FSM | P1 (KL-13) |
| **ArrayOpCodeGen** | `hw.array_inject/get` 코드 생성 | P2 (KL-16) |
| **CombinationalLowering** | `llhd.combinational` → `comb.*` | P2 (KL-11) |
| **AddressDecodingAnalysis** | 조합 로직에서 레지스터 오프셋 추출 | P2 |
| **RegisterBehaviorAnalysis** | 레지스터 읽기/쓰기 동작 추출 | P2 |
| **ProtocolRecognition** | 포트 패턴에서 APB/AXI/AHB 인식 | P3 |

## 4. 미결 결정

### D1: GenRAL 접근 방식

현행: IR-only (레지스터 추출, 필드/접근 권한 불가)
**권고**: 하이브리드 (기본=IR-only, `--ral-rdl` 옵션으로 SystemRDL 보강)
→ 구현: Phase 3+

### D2: GenFuncModel 재설계 범위

현행: FSM 추출만 구현됨
**권고**: (1) 레지스터 인터페이스 추출 우선 → (2) FSM 인식 → (3) 제어 흐름 추상화
→ AddressDecodingAnalysis + RegisterBehaviorAnalysis 구현 후 재착수

### D3: GenModel IR 진입 조건 공식화 (미착수)

**내용**: 현재 단일 체크(`llhd.prb/drv`)만 존재. `iN(N>64)`, `hw.array_inject`, `llhd.combinational` 체크 추가 + 공식 precondition 문서화 필요.
→ 다음 세션 브레인스토밍 주제 (`docs/plans/2026-03-06-genmodel-ir-spec-brainstorm.md` 참조)
